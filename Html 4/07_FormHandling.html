Form Handling

Forms are an important part of web development as they provide a way for users to interact with the website and send data to the server. The data sent by the form can be used to update a database, send an email, or perform any other action on the server. The method attribute specifies how the form data should be sent to the server. The two most commonly used methods are GET and POST.

The GET method sends the form data as part of the URL in name-value pairs. The data is visible in the URL and is limited in size. It is used for non-sensitive data and bookmarkable data. The data is also cached by the browser and is not suitable for sensitive data. The GET method is also used for search forms and filters.

The POST method sends the form data as part of the body of the HTTP request. The data is not visible in the URL and the size of the data is unlimited. It is used for sensitive data and for actions that should not be bookmarkable. The POST method is also used for creating new records in a database.

The <form> tag is used to create a form in the document. It has the following attributes:

  • action - specifies the URL of the form-data receiver
  • method - specifies the HTTP method to use when submitting the form - GET or POST
  • enctype - specifies the type of encoding to use when submitting the form - application/x-www-form-urlencoded, multipart/form-data, or text/plain
  • name - specifies the name of the form
  • target - specifies where to open the response received after submitting the form - _blank, _self, _parent, or _top

The form above uses several HTML tags:

  • <form>: Defines an HTML form for user input.
  • <input>: Defines an input control. Types used include:
    • text: Used for a single-line text input.
    • email: Used for an input field for email addresses.
    • radio: Used for selecting one option from a set.
    • submit: Defines a button for submitting form data.
  • <label>: Defines a label for an <input> element.
  • <select>: Creates a drop-down list.
  • <option>: Defines an option in a drop-down list.
  • <table>, <tr>, <td>: Used to organize the form layout in a tabular format.

The <optgroup> tag is used to group related options in a <select> menu. This is useful for categorizing options under a single heading, improving the organization and readability of the drop-down list for users.

The <form> tag has the following parameters:

  • accept - specifies a comma-separated list of MIME types a server can handle
  • accept-charset - specifies a character encoding that must be accepted by the server
  • autocomplete - specifies whether a form should have autocomplete enabled or disabled
  • novalidate - specifies that the form should not be validated when submitted
